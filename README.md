# CropSeek LLM

### AI-Powered Agriculture Decision Support System

CropSeek LLM is a production-grade backend API that helps farmers make data-driven decisions using a combination of **Machine Learning**, **Deep Learning**, **Large Language Models (LLMs)**, and **Live Weather Intelligence**.

It is not a simple chatbot. It is a full AI pipeline where a conversational interface (powered by Google Gemini) intelligently routes farmer queries to specialized ML models, executes them, and returns farmer-friendly explanations — all in real time.

---

## What It Does

| Feature | Description |
|---|---|
|  Crop Recommendation | Recommends the best crop based on soil NPK, pH, temperature, humidity, rainfall, and season |
|  Fertilizer Recommendation | Suggests the optimal fertilizer based on soil type, crop, and nutrient levels |
|  Disease Detection | Identifies plant diseases from leaf photos using a deep learning CNN |
|  Live Weather Integration | Auto-fetches real-time weather by city name via OpenWeatherMap |
|  AI Chat Assistant | Natural language interface powered by Google Gemini 2.5 Flash |
|  Multilingual Support | Responds in English, Hindi, Telugu, Tamil, and Kannada |
|  Session Memory | Remembers conversation history per user session |

## Installation

### Prerequisites

- Python 3.9 or higher
- pip

### Step 1 — Clone the repository

```bash
git clone https://github.com/your-username/cropseek-llm.git
cd cropseek-llm
```

### Step 2 — Create and activate virtual environment

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 4 — Set up environment variables

```bash
cp env.example .env
```

Open `.env` and fill in your API keys:

```env
# LLM Provider
LLM_PROVIDER=gemini

# Google Gemini (get free key at https://aistudio.google.com/app/apikey)
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.5-flash

# OpenWeatherMap (get free key at https://openweathermap.org/api)
WEATHER_API_KEY=your_openweather_api_key_here
```

### Step 5 — Train the ML models

```bash
python scripts/train_models.py
```

This will:
- Auto-download the crop and fertilizer datasets if not present
- Train the stacked ensemble crop model (takes ~2–3 minutes)
- Train the RandomForest fertilizer model
- Save all models to the `models/` directory

> **Note:** `models/crop_model.pkl` (2.8 GB) is excluded from the repository due to GitHub's file size limit. You must run this script to generate it locally.

### Step 6 — Download the disease detection model

```bash
python scripts/download_disease_model.py
```

This downloads a pre-trained MobileNetV2 model (~9.3 MB) from HuggingFace. No Kaggle account or GPU required.

### Step 7 — Start the server

```bash
python main.py
```

The server starts at `http://localhost:8000`

### POST `/api/chat`
Natural language agricultural assistant powered by Gemini AI.

**Request Body:**
```json
{
  "query": "My soil has N=90 P=42 K=43 ph=6.5. What crop should I grow?",
  "session_id": "user-abc-123"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| query | string | ✅ | Natural language question (max 2000 chars) |
| session_id | string | ❌ | Unique ID for conversation memory |

**Rate limit:** 20 requests per minute per IP address.

**Response:**
```json
{
  "success": true,
  "session_id": "user-abc-123",
  "intent": "recommend_crop",
  "tool_used": "recommend_crop",
  "llm_mode": "gemini_tool_call",
  "explanation": "Based on your soil conditions, **Rice** is the best crop for you right now!...",
  "next_action": "Next: Get fertilizer recommendations for rice.",
```

`llm_mode` values:
- `gemini_tool_call` — Gemini called an ML tool and returned a result
- `gemini_text` — Gemini responded in plain text (no tool needed)
- `mock_rule_based` — Rule-based fallback (no LLM configured)
- `mock_fallback` — LLM failed, fell back to rules

## How the AI Pipeline Works

```
User sends natural language query
            │
            ▼
    Gemini 2.5 Flash reads the message
    + system prompt with tool definitions
            │
            ├─── Soil/weather data detected?
            │         └── outputs JSON: {"tool":"recommend_crop","parameters":{...}}
            │
            ├─── Fertilizer question?
            │         └── outputs JSON: {"tool":"recommend_fertilizer","parameters":{...}}
            │
            ├─── Location given, no weather?
            │         └── outputs JSON: {"tool":"get_weather","parameters":{"city":"..."}}
            │
            └─── General question?
                      └── responds in plain text
            │
            ▼
    Engine parses JSON, calls the ML tool
            │
            ▼
    Tool result sent back to Gemini
            │
            ▼
    Gemini writes farmer-friendly explanation
    in the user's detected language
            │
            ▼
    Response returned to client
```

## ML Models — Technical Details

### Crop Recommendation Model

**Architecture:** Out-of-fold stacking ensemble

```
Base models (5-fold cross-validation):
  ├── RandomForestClassifier     (300 trees)
  ├── ExtraTreesClassifier       (300 trees)
  └── GradientBoostingClassifier (200 trees, lr=0.05)
          │
          ▼
  Meta-learner: LogisticRegression
```

**Scoring formula:**
```
composite_score = (0.55 × ML_probability)
                + (0.25 × seasonal_score)
                + (0.20 × weather_score)
```

**Seasonal scoring:** Each crop has a defined season (Kharif/Rabi/Zaid) and sowing months. The seasonal score is 1.0 if the current month matches, decreasing as the month moves away from the optimal sowing window.

**Uncertainty detection:** Shannon entropy of the probability distribution. If entropy > 1.8, the model is considered uncertain and falls back to an extended RandomForest model trained on 102 crops.

**Training data:** `Crop_recommendation.csv` — 2200 samples, 22 crop classes, 7 features (N, P, K, temperature, humidity, pH, rainfall)

**Accuracy:** 100% top-1, 100% top-3 on training set. High generalization due to stacking.

---

### Fertilizer Recommendation Model

**Architecture:** RandomForestClassifier (300 trees)

**Features:** Temperature, Humidity, Moisture, Soil Type (encoded), Crop Type (encoded), Nitrogen, Potassium, Phosphorous

**Rule-based overrides (applied before ML):**

| Condition | Override Fertilizer | Reason |
|---|---|---|
| Nitrogen < 20 | Urea | Critical nitrogen deficiency |
| Phosphorus < 10 | DAP | Critical phosphorus deficiency |
| Potassium < 10 | MOP | Critical potassium deficiency |
| N < 40 and P < 20 | NPK (20-20-0) | Low N and P |
| N < 40 and K < 20 | NPK (17-17-17) | Low N and K |

**Accuracy:** 95% on test set

---

### Disease Detection Model

**Architecture:** MobileNetV2 (pre-trained on ImageNet, fine-tuned on PlantVillage)

**Source:** [Daksh159/plant-disease-mobilenetv2](https://huggingface.co/Daksh159/plant-disease-mobilenetv2) on HuggingFace

**Training data:** PlantVillage augmented dataset — 87,000 images, 38 classes

**Input:** 224×224 RGB image, normalized with ImageNet mean/std

**Classifier head:**
```python
model.classifier[1] = nn.Sequential(
    nn.Dropout(0.2),
    nn.Linear(1280, 38)
)
```

**Accuracy:** ~95% on PlantVillage validation set

**Supported crops and diseases (38 classes):**

| Crop | Diseases |
|---|---|
| Apple | Scab, Black Rot, Cedar Apple Rust, Healthy |
| Blueberry | Healthy |
| Cherry | Powdery Mildew, Healthy |
| Corn | Cercospora Leaf Spot, Common Rust, Northern Leaf Blight, Healthy |
| Grape | Black Rot, Esca, Leaf Blight, Healthy |
| Orange | Haunglongbing (Citrus Greening) |
| Peach | Bacterial Spot, Healthy |
| Pepper | Bacterial Spot, Healthy |
| Potato | Early Blight, Late Blight, Healthy |
| Raspberry | Healthy |
| Soybean | Healthy |
| Squash | Powdery Mildew |
| Strawberry | Leaf Scorch, Healthy |
| Tomato | Bacterial Spot, Early Blight, Late Blight, Leaf Mold, Septoria Leaf Spot, Spider Mites, Target Spot, Yellow Leaf Curl Virus, Mosaic Virus, Healthy |

## Fallback Chain

The system never crashes. If any component fails, it gracefully degrades:

```
Gemini fails → falls back to mock rule-based
Weather API fails → uses manually provided values
Disease model missing → returns clear error message
Fertilizer model missing → returns clear error message
Crop model missing → returns clear error message
Invalid input → returns validation error with field details
```
